HORIZONTAL PE
IDOR — Access Another User's Profile
Alice (id=2) fetches another user's profile by changing the ?id= query param. In vulnerable mode, no ownership check is done.
Target: GET /profile?id={id}
VERTICAL PE
Admin Panel — No Role Check
A normal user accesses /admin. In vulnerable mode, there's no authentication or role check — any user gets admin access.
Target: GET /admin
📖
How It Works
The vulnerability and the fix
1
IDOR (Insecure Direct Object Reference)
The server uses a user-controlled id param with no ownership check. Alice can access Bob's data just by changing the number.
2
Vertical Privilege Escalation
The /admin route has no authentication middleware — any request, even without a session, gets admin content.
3
IDOR Fix — Ownership Enforcement
In fixed mode, the server checks session.user.id === requestedId before returning profile data. Non-admins get 403.
4
Admin Fix — RBAC
In fixed mode, the server checks session.user.role === 'admin'. Regular users get a 403 Forbidden response.
